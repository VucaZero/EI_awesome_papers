# TEMPLATES

本文档规定仓库统一使用的笔记模板、文件命名规则与提交最小要求。

## 1. 文件命名规范

统一命名格式：

```text
<type>__<topic>__<year>__<first_author>__<short_title>.md
```

说明：

- `<type>` 取值：`paper`、`survey`、`theory`、`analysis`
- `<topic>` 取值：`VLA`、`VLN`、`EQA`、`MANI`、`AFF`、`WM`、`BENCH`、`SURV`
- `<year>` 使用 4 位年份
- `<first_author>` 使用第一作者姓氏的小写英文
- `<short_title>` 使用 2 到 6 个英文单词的缩写表达

示例：

```text
paper__VLA__2025__zhou__openvla.md
survey__SURV__2024__wang__embodied_ai_survey.md
theory__WM__2023__li__latent_planning.md
analysis__MANI__2026__group__mobile_manip_failure_modes.md
```

## 2. 存放规则

- 单篇论文笔记放入对应主题目录的 `paper_notes/`
- 主题整理、阶段总结、组内分析放入对应主题目录的 `topic_summaries/`
- 一篇文章如果横跨多个主题，主文件放在最相关主题目录，其他目录用 `README.md` 记录交叉引用

## 3. 单篇论文总结模板

适用于新论文精读、复现前的快速理解与组会汇报准备。

```md
# 论文信息
- 标题：
- 作者：
- 年份 / 会议：
- 论文链接：
- 代码链接：

# Q1. 一句话总结挑战

# Q2. 一句话描述技术路径

# Q3. 这篇工作解决了什么问题，为什么这个问题重要

# Q4. 方法的核心模块是什么，真正起作用的机制可能是什么

# Q5. 实验设置、数据集、指标与最关键结果是什么

# Q6. 局限性、失败案例与可能的改进方向

# Q7. 和我们关注方向的关系
```

## 4. 综述总结模板

适用于 survey、tutorial、position paper。

```md
# 文献信息
- 标题：
- 作者：
- 年份 / 来源：
- 链接：

# Q1. 这篇综述试图统一回答什么核心问题

# Q2. 它给出的分类框架是什么

# Q3. 每个类别的代表方法与差异是什么

# Q4. 它如何总结当前瓶颈、争议点与开放问题

# Q5. 哪些 benchmark、环境、数据集最值得优先关注

# Q6. 对我们当前研究选题最有价值的启发是什么
```

## 5. 理论文章简介模板

适用于偏理论、偏建模、偏分析的文章。

```md
# 文献信息
- 标题：
- 作者：
- 年份 / 来源：
- 链接：

# Q1. 一句话说明作者想解释或证明什么

# Q2. 文章采用了什么理论框架或形式化建模方式

# Q3. 最关键的定义、命题或推导链条是什么

# Q4. 这个理论结论成立依赖哪些假设

# Q5. 这个理论和实际具身任务之间的映射关系是什么

# Q6. 对我们做实验、建模或分析有什么直接帮助
```

## 6. 主题整理 / 分析模板

适用于阶段性读书总结、方向地图、失败模式分析、方法对比。

```md
# 主题信息
- 主题：
- 时间范围：
- 覆盖论文：

# Q1. 这个主题当前的核心挑战是什么

# Q2. 主流技术路线可以如何分组

# Q3. 各路线默认依赖了哪些数据、环境或先验

# Q4. 哪些实验设置让结果看起来更强，但未必公平

# Q5. 目前最明显的能力短板与空白问题是什么

# Q6. 对我们下一步最值得投入的研究切入点是什么
```

## 7. 最小提交要求

每一篇正式提交到仓库的笔记，至少应满足：

- 有论文基本信息和链接
- 有一句话挑战总结
- 有一句话技术路径总结
- 有实验环境 / 数据集 / 指标记录
- 有与本组研究关系的说明

## 8. 写作建议

- 优先写“为什么有效”，不要只抄方法结构图
- 优先写“实验是否真的支持结论”，不要只抄表格
- 尽量写出作者隐含假设与可能的失败边界
- 避免大段原文翻译，尽量压缩成可检索、可复用的判断
